OscMachine triggering samples with OSC messages


Inherits from: Object


OscMachine is a sampler, that uses OSC messages as trigger. It has a variable number 

of tracks and you can define on what messages the particular tracks should react.

OscMachine uses heavily the RedSampler by redFrik.


See also: Object, RedSampler, OSCresponderNode, SoundFile, SoundFileView


Some Important Issues Regarding OscMachine 


The port for incoming OSC messages is 57120. This is hardcoded an you can not change it. 

That's why it can not be used with SwingOSC on Linux. SwingOSC uses this port for ping.

For more information see OSC_communication and SwingOSC.


Also very important to boot the server (!!!) before running OscMachine.


Creation / Class Methods


*new (trackNumber, server, diskPlay, showWin, overlap)

Creates a new OscMachine.

trackNumber - Number of tracks. Default value is 1.

server - You can specify the server the files should play on. Default value is nil,

which starts the local server.

diskPlay - If this is true samples are played from disk and not from memory. 

Default value is false. Not implemented yet completely.

showWin - If true you get a gui. Default value is true.

overlap - Setting maximum number of overlaps for one sample. Default value is 30.

// inline example

s.boot;

g = OscMachine.new;

g.close;

or

s.boot;

g = OscMachine.new(8);

g.close;


Accessing Instance and Class Variables

diskPlay

If true samples are played from disk, not from ram.

Default value is false.

debugMode_(arg1)

debugMode

If true, you get information about values in post window.

Default value is true.

Doing Some Task (optional)


A short bit of prose explaining something about the task.

close

Clean up method. Deletes the OSCresponder nodes.

// inline example

s.boot;

g = OscMachine.new(showWin: false);

g.close; // deletes the responders



showWindow

Short prose description of method.

// inline example

s.boot;

g = OscMachine.new(showWin: false);

g.showWindow; // explanation

g.close;



hideWindow

Hides the GUI window. If you want to close OscMachine you have to call close method.

// inline example

s.boot;

g = OscMachine.new;

g.hideWindow;

g.close;



setSampleFile (pos, samplePath)

Sets the sample file for a given track.

pos - Track number.

samplePath - Path to sample file.


playSample (pos)

Play sample on given track.

pos - Track number.


setOscMsg (number, msg)

Sets the OSC trigger message on a specific track and creates OSCresponder.

number - Track number.

msg - Array with OSC messages.

// inline example

a = OscMachine.new;

a.setSampleFile(0,"/Users/peter/scwork/snd/AltoSax.aif");

a.setOscMsg(0,["/play", "true", 1]);



setResponder (pos)

Creates OSCresponderNode.

pos - Track number.



Examples


// creating OscMacines with different track number.

OscMachine.new

OscMachine.new(10)

OscMachine.new(diskPlay: true);


// One track and three messages.

(

a = OscMachine.new;

a.setSampleFile(0,"/Users/peter/scwork/snd/AltoSax.aif");

a.setOscMsg(0,["/play", "true", 1]);

)


// Two tracks.

(

a = OscMachine.new(2);

a.setSampleFile(0,"/Users/peter/scwork/snd/AltoSax.aif");

a.setOscMsg(0,["/play", 0]);

a.setSampleFile(1,"/Users/peter/scwork/snd/axaxaxas.aiff");

a.setOscMsg(1,["/play", 1]);

//a.debugMode = false;

)


// with efx

(

a = OscMachine.new(2);

a.setSampleFile(0,"/Users/peter/scwork/snd/AltoSax.aif");

a.setOscMsg(0,["/play", 0], ["positionx"]);

a.setSampleFile(1,"/Users/peter/scwork/snd/axaxaxas.aiff");

a.setOscMsg(1,["/play", 1]);

//a.debugMode = false;

)



// no window mode

(

a = OscMachine.new(2, showWin: false);

a.setSampleFile(0,"/Users/peter/scwork/snd/AltoSax.aif");

a.setOscMsg(0,["/play", 0]);

a.setSampleFile(1,"/Users/peter/scwork/snd/axaxaxas.aiff");

a.setOscMsg(1,["/play", 1]);

//a.debugMode = false;

)


a.showWindow;

a.hideWindow;

// if window is not used you have to call close fuction,

// else OSCresponderNodes are still remaining "alive".

a.close;
